## מה בוצע בפועל
1. הכלי קרא את `App.java` ואת `AppTest.java` הקיים, וזיהה לכל מתודה את הנתיבים הלוגיים לכיסוי.
2. נכתב מחדש `AppTest.java` עם 11 מחלקות `@Nested` (אחת לכל מתודה), שימוש מגוון ב‑assertions,
   ו‑`@ParameterizedTest` ל‑`isPrime` ו‑`factorial`.
3. הורצה הפקודה `./gradlew clean test` – **56 בדיקות עברו, 0 כשלונות**.
4. בוצע מעבר ידני מתודה‑מתודה לאימות שאין ענף שאינו מכוסה.
5. נוסף תוסף **JaCoCo** ל‑`build.gradle.kts`, והורץ `./gradlew clean test jacocoTestReport` כדי
   לאמת את הכיסוי אובייקטיבית: **100% בכל המדדים** (Branch 26/26, Line 46/46, Method 12/12,
   Instruction 277/277). נוספה בדיקה אחת (`assertNotNull(new App())`) שמכסה גם את ה‑constructor
   הדיפולטי.

## איך יודעים שהבדיקות מספיקות (הסיכום המקצועי מהשיחה)
- **סוגי כיסוי:** Statement < Branch < Path. היעד המעשי הוא **branch coverage** גבוה (כל ענף של
  כל `if`/לולאה נבדק ל‑true ו‑false).
- **בניית קלטים:** Equivalence Partitioning (נציג מכל מחלקת שקילות) + Boundary Value Analysis
  (בדיקה בדיוק על הגבולות, שם מתחבאים באגים).
- **אימות בפועל:** מעבר ידני על כל ענף, וכלי מדידה אוטומטי **JaCoCo**
  (`./gradlew test jacocoTestReport`) שמראה אילו שורות/ענפים לא כוסו. בפרויקט זה התוצאה היא
  **100% כיסוי ענפים (Branch) ושורות (Line)** – כלומר כל הנתיבים נבדקו בפועל.
- **חוזק הבדיקות:** **Mutation Testing (PIT)** – מכניס מוטציות לקוד ובודק אם בדיקה נכשלת; מוטציה
  ש"שורדת" מסמנת חור בבדיקות.

## טבלת כיסוי הנתיבים (אימות סופי)
| מתודה | נתיבים שכוסו |
|---|---|
| `add` | חיובי/שלילי/אפס, קומוטטיביות, גלישת `int` (overflow) |
| `isPrime` | `n<2` (כולל שליליים), גבול 2/3 (לולאה לא נכנסת), נמצא מחלק (15,25), לא נמצא (13,97) |
| `reverse` | רגיל, ריק, תו בודד, היפוך כפול=מקור, `null`→`NullPointerException` |
| `factorial` | `n<0`→חריגה+הודעה, גבול `n=0/1`, ערכים רגילים, `assertDoesNotThrow` |
| `isPalindrome` | `true` (רישיות/פיסוק/רווחים), `false`, ריק / רק‑פיסוק |
| `fibonacciUpTo` | רצף תקין, `n<0`→חריגה, גבולות `n=0`/`n=1`, עצירה כש‑13>10 |
| `charFrequency` | ספירה case‑sensitive, מחרוזת ריקה→מפה ריקה, תו חסר→`null` |
| `isAnagram` | אנגרמה תקינה (רישיות+רווחים), אורך שונה, תווים שונים |
| `average` | מערך רגיל, שליליים, מערך ריק→חריגה+הודעה |
| `filterEvens` | זוגיים כולל 0/שליליים, רשימה ריקה, ללא זוגיים, כולם‑זוגיים |
| `mostCommonWord` | מילה שכיחה, התעלמות מפיסוק, מילה בודדת, אימות lower‑case |

**מסקנה:** כל ענף לוגי בכל אחת מ‑11 המתודות מכוסה לפחות בבדיקה אחת, כולל כל החריגות וכל מקרי הקצה.
