**1. סוגי כיסוי (Coverage) – מהחלש לחזק:**
- **Statement coverage** – האם כל שורת קוד הורצה לפחות פעם אחת.
- **Branch coverage** – האם כל ענף של כל `if`/לולאה נבדק גם כש‑true וגם כש‑false. זה היעד המעשי שאליו נשאף.
- **Path coverage** – כל **שילוב** של ענפים. בפונקציות עם הרבה תנאים זה מתפוצץ קומבינטורית, אז בפועל מסתפקים ב‑branch coverage טוב.

**2. איך בונים קלטים שמכסים נתיבים – שתי טכניקות קלאסיות:**
- **Equivalence Partitioning** – מחלקים את מרחב הקלט ל"מחלקות שקילות" שמתנהגות אותו דבר, ובוחרים נציג אחד מכל מחלקה. ל‑`factorial`: `{n<0}`, `{n=0,1}`, `{n>1}`.
- **Boundary Value Analysis** – הבאגים אוהבים להתחבא בגבולות, אז בודקים בדיוק על הקצה. ל‑`isPrime` הגבול הוא 2 (הראשוני הקטן ביותר) ו‑1 (ממש מתחת לסף).

**3. איך מוודאים בפועל שלא פספסנו – שתי שיטות:**
- **מעבר ידני על כל ענף** – לקרוא כל מתודה ולסמן: לכל `if`/`throw`/לולאה, יש לי בדיקה ל‑true ול‑false? בוא אדגים על `isPrime`:
  ```
  if (n < 2) return false;        // �o ענף true: isPrime(1),(0),(-7)  ✓
                                  // ✓ ענף false: ממשיך הלאה
  for (i=2; i<=sqrt(n); i++)      // ✓ לולאה לא נכנסת: isPrime(2),(3)
      if (n % i == 0) return false;// ✓ נמצא מחלק: isPrime(15),(25)
  return true;                    // ✓ לא נמצא מחלק: isPrime(13),(97)
  ```
  כל ארבעת הנתיבים מכוסים.
- **כלי מדידה אוטומטי – JaCoCo** (ספריית open‑source חינמית ל‑Java שמודדת אוטומטית אחוז כיסוי קוד – כלומר אילו שורות וענפים בקוד באמת הורצו על‑ידי הבדיקות ואילו לא). מוסיפים את הפלאגין ל‑`build.gradle.kts`:
  ```kotlin
  plugins { jacoco }
  tasks.test { finalizedBy(tasks.jacocoTestReport) }
  ```
  מריצים `./gradlew test jacocoTestReport` והדוח ב‑`app/build/reports/jacoco/test/html/index.html` מראה באדום בדיוק אילו שורות/ענפים **לא** כוסו. זה ה"מד אובייקטיבי".

**4. מעבר לכיסוי – האם הבדיקות עצמן חזקות? Mutation Testing (PIT):**
כיסוי 100% עדיין יכול להיות חלש אם ה‑assertions לא באמת בודקים. PIT מכניס "מוטציות" לקוד (למשל הופך `<` ל‑`<=`) ובודק אם בדיקה כלשהי נכשלת. אם המוטציה "שורדת" – יש לך חור בבדיקות. זה ה‑gold standard לאיכות בדיקות.

לסיכום – הבדיקות "מספיקות" כש: (א) כל ענף מכוסה (branch coverage גבוה, נמדד ב‑JaCoCo), (ב) כל מקרה קצה וחריגה נבדקו, ו‑(ג) המוטציות לא שורדות (PIT). רוצה שאעבור מתודה‑מתודה ואוודא שאין ענף לא מכוסה בקובץ הנוכחי?
